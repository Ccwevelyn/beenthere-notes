#### Advantage/ Disadvantage
##### 1. **Scatter Chart**
**Advantages**

+ Excellent for detecting relationships, correlations, and patterns 擅长展示变量之间的关系与趋势
+ Trendline helps show **overall direction** 趋势线能显示总体方向
+ Good for identifying **outliers** 能识别异常点

**Disadvantages**

+ Only compares **two** variables 
+ Hard to interpret if points overlap 点重叠

**When to Choose**

+ 研究变量之间的相关性 correlation
+ When analyzing **continuous quantitative** data 分析连续型定量数据时

#####  2. **Scatter‑Chart Matrix（散点图矩阵）**
A grid of scatter charts comparing every pair of variables. 将所有变量两两组合成散点图矩阵。

**Adv****antages**

+ Shows relationships among **multiple** variables 可同时观察**多个变量之间的关系**
+ Useful for exploratory data analysis (**EDA**) 在数据探索阶段非常有用，帮助你快速判断哪些变量值得进一步建模。

**Disadvantages**

+ Many small charts may overwhelm readers 小图太多可能造成信息过载
+  Not suitable for categorical variables  

**When to Choose**

+ Analyzing datasets with many quantitative variables 多变量数据分析时

##### 3. **Line Chart（折线图）**
Connects data points with a line; ideal for time‑series. 用折线连接数据点，适合时间序列。

**Advantages**

+ Best for showing trends <font style="color:#DF2A3F;">over time </font>最适合展示随时间变化的趋势
+ Easy to compare multiple lines 易于比较多条趋势线

**Disadvantages**

+ Too many lines cause clutter 线太多会变得拥挤
+ Not suitable for categorical comparisons 不适合比较类别数据

**When to Choose**

+ When analyzing time‑series data 分析时间序列数据时

##### 4. **Sparkline（迷你折线图）**
**Advantages**

+ Takes very little space 占用空间极小
+ Shows trend clearly 清晰展示趋势
+ No axes  very clean 无坐标轴，视觉简洁

**Disadvantages**

+ Cannot show exact values 无法显示精确数值
+ Not suitable for detailed analysis 不适合深入分析

**When to Choose**

+ When you need compact trend indicators 需要紧凑趋势指示器时
+ When displaying many small trends 展示多个小趋势时

##### 5. **Heat Map（热力图）**
Uses color intensity to represent magnitude. 用颜色深浅表示数值大小。

**Advantages**

+ Good for comparing across areas or time 适合区域或时间比较
+ Excellent for identifying **patterns, clusters, and anomalies** 擅长识别模式、聚集区域和异常点
    - 哪些区域数值特别高（hot spots）
    - 哪些区域特别低（cold spots）

**Disadvantages**

+ Hard to read exact values 难以读取精确数值
+ Color perception varies 不同读者对颜色敏感度不同

**When to Choose**

+ When showing intensity or density 展示强度或密度时

##### 6. **Parallel‑Coordinates Plot（平行坐标图）**
**Advantages**

+ Excellent for high‑dimensional data 
+ Shows patterns, clusters, and outliers 能展示模式、聚类与异常点
    - 通过将每条观测记录在所有坐标轴上连接成折线，平行坐标图能够展示多维数据中的模式：
        * 多条线走向相似 → 聚类
        * 某些线明显偏离 → 异常点
        * 多维趋势 → 变量之间的整体关系
+ Allows comparison across many variables 

**Disadvantages**

+ Lines may overlap heavily 线条可能严重重叠
+ Hard for beginners 初学者难以理解

**When to Choose**

+ When analyzing datasets with many variables 多变量数据分析时
+ When exploring patterns across dimensions 想探索多维模式时

##### 7. **Treemap（树图）**
**Advantages**

+ Great for showing proportions  树图通过矩形面积表示类别大小，非常适合展示层级结构中的比例关系。
+ Efficient use of space  树图能充分利用整个绘图区域，不会留下空白，非常适合在有限空间中展示大量类别  

**Disadvantages**

+ Hard to compare similar‑sized rectangles 相近大小难比较
+ Not good for trends 不适合展示趋势

**When to Choose**

+ When showing hierarchical structure 展示层级结构时
+ When comparing proportions 比较比例时

##### 8. **GIS Chart（地理信息系统图表）**
**Advantages**

+ Shows spatial patterns clearly 清晰展示空间模式
+ Helps identify clusters and hotspots 有助识别聚集区域
+ Useful for market targeting 适用于市场定位

**Disadvantages**

+ Requires geographic data 需要地理数据
+ Dense areas may overlap 密集区域可能重叠

**When to Choose**

+ When analyzing geographic distribution 分析地理分布时
+ When identifying regional markets 寻找区域市场时

### <font style="color:rgb(15,17,21);">Steps of Hypothesis Testing </font>
假设检验是一种统计方法，用来判断：** 样本数据**是否提供了**足够的证据**来支持对总体的某个主张。  

#### Overview
##### Data Sets：Population, Sample, Parameter, Statistic  
**Population**（总体）是指我们感兴趣的**所有**结果、所有回答、所有测量值或所有计数的集合。

**Parameter**（参数）**描述****<u>总体</u>****特征的数值**。

+ 如，**全**美国所有人的真实平均年龄

**Sample**（样本）样本是从总体中抽取出来的**一部分**。 

**Statistic**（统计量）**描述****<u>样本</u>****特征的数值**。 

+ 如，从三个州**抽样**得到的一组人，他们的样本平均年龄

参数通常是我们想知道但不知道的东西，只能通过样本来推断。

统计量是我们**从样本算出来的**，用来估计总体的参数。

#####  Hypothesis Testing：Null & Alternative Hypotheses 原假设 & 备择假设 
###### What is hypothesis testing?（什么是假设检验）
We make a **tentative conjecture** about a population parameter and then use sample data to test whether this conjecture is reasonable.

我们先对总体参数提出一个**暂时性的猜测**，然后用样本数据来检验这个猜测是否合理。

###### Null Hypothesis H₀（原假设）
The null hypothesis H₀ is the **default claim** about the population parameter. 

原假设 H₀ 是关于总体参数的**默认说法**

Often it represents **"no effect," "no difference**," or "**no improvement."  **

###### Alternative Hypothesis Hₐ（备择假设）
The alternative hypothesis Hₐ is the **opposite** of the null hypothesis. It is often the **research hypothesis** that the researcher hopes to support.

备择假设 Hₐ 是原假设的**对立面**，通常就是研究者<u>真正想要证明</u>的**研究假设**。

#### Steps 
<!-- 这是一张图片，ocr 内容为： -->
![](https://cdn.nlark.com/yuque/0/2026/png/42556572/1775473690264-c347b17b-01b8-4b48-985b-3011185c10b5.png)

##### Step 1. Develop the null and alternative hypotheses
**建立原假设和备择假设**

Translate the **real‑world question**into two mathematical statements about a **population parameter**(usually μ or p):

+ Null hypothesis H<sub>0</sub>: the **default claim** about the **population**
+ Alternative hypothesis H<sub>a</sub>: the research claim we want evidence for

如，

1. 先写H<sub>a</sub> -->  想证明的结论

“Does the new fuel injection system **increase** the average miles per gallon above 24?”  

 因为研究假设是“**大于 24**”，所以备择假设写成：  H<sub>a</sub>: μ>24 

2.  再写“默认不成立的版本”  

 The null hypothesis is the **opposite** or **complement** of Hₐ in terms of the parameter:  H<sub>0</sub>: μ≤24

##### Step 2. Specify the level of significance (α)
**设定显著性水平 ****α**

<!-- 这是一张图片，ocr 内容为： -->
![](https://cdn.nlark.com/yuque/0/2026/png/42556572/1775474456875-f89fb3f8-1375-447a-9f91-1a0f5fdb8b47.png)

| 真实情况 | 你的结论 | 结果 |
| --- | --- | --- |
| H₀ True | Reject H₀ | Type I error（错杀好人） |
| H₀ False | Do not reject H₀ | Type II error（放走坏人） |


α 就是你事先设定的容错率。

##### Step 3. Collect the sample data and compute the value of the test statistic
收集样本数据并计算检验统计量

##### Step 4. Use the value of the test statistic to compute the p‑value
用检验统计量计算 p 值

##### Step 5. Reject H<sub>0</sub> if p≤α
如果 p ≤ α，则拒绝 H₀

##### Step 6. Interpret the statistical conclusion in the context of the application
在实际情境中解释统计结论

把“拒绝 / 不拒绝 H₀”的统计结论，翻译回**现实世界的说法**。

### <font style="color:rgb(15,17,21);">Multicollinearity: Problems and Solutions </font>
 （多重共线性）  

**<font style="color:rgb(0,0,0);">Multicollinearity </font>**<font style="color:rgb(0,0,0);">refers to the correlation among the independent variables in </font>**<font style="color:rgb(255,0,0);">multiple regression analysis</font>**<font style="color:rgb(0,0,0);">. </font>多个自变量之间高度相关（彼此相似、重复解释）

<font style="color:rgb(0,0,0);"> X₁ 和 X₂ 之间的关系太强，它们几乎提供了同样的信息。  </font>

#### <font style="color:rgb(0,0,0);">Problem</font>
+ **<u>Problem</u>**:  It is possible to conclude that a parameter is not significantly different from zero when the variable actually has a strong relationship with the dependent variable    多重共线性会让一个本来很重要的变量，看起来“不显著”。

X₁ 和 X₂ 太像, 模型无法区分“到底是谁在解释 Y”, 导致：

    - 系数变得不稳定
    - 标准误差变大
    - p 值变大
    - 看起来“不显著”  

####  Rule of Thumb  
 如果两个自变量的相关系数 |r| > 0.7，就可能有多重共线性问题。  

+ | r | > 0.7 → 强相关
+ | r | > 0.8 → 很强相关
+ | r | > 0.9 → 几乎重复变量

####  Solutions  
##### **Remove highly correlated variables**
**删除高度相关的自变量**

如果两个自变量之间的相关系数 |r| > 0.7，则它们提供的是重复信息，通常只保留一个即可。 **目的：减少重复解释 Y 的信息。**

##### **Combine variables into a single measure**
**将高度相关的变量合并成一个指标**

当两个变量都重要但高度相关时，可以合并成一个新的变量（如总费用、指数、比率）。 **目的：保留信息但减少共线性。**

##### **Use variable selection procedures**
**使用变量选择方法**

Backward elimination、Forward selection、Stepwise selection 这些方法会自动剔除冗余或不显著的变量。 **目的：让模型更简洁、更稳定。**

##### **Apply dimensionality reduction (e.g., PCA)**
**使用降维方法（如主成分分析 PCA）**

将多个相关变量压缩成少数几个“主成分”，再用主成分做回归。 **目的：从根本上解决共线性。**

##### **Increase sample size**
**增加样本量**

样本越多，标准误差越小，共线性带来的不稳定性越弱。 **目的：减少估计波动，提高系数稳定性。**

#### Linear regression
1. **加载包、建立简单线性回归模型**

```r
install.packages("lmtest")
library(lmtest)
model <- lm(FinalExam ~ Assignment, data = finalscore)

# model ouput
summary(model)
```

+ `install.packages("lmtest")`：安装 `lmtest` 包（只需要装一次）。
+ `library(lmtest)`：加载包，后面要用里面的检验函数（比如 `bptest`）。
+ `lm(FinalExam ~ Assignment, data = finalscore)`：建立**简单线性回归模型**，因变量是 `FinalExam`，自变量是 `Assignment`。
+ `summary(model)`：输出模型结果，包括：
    - 回归系数（截距、斜率）
    - 每个系数的 t 值、p 值
    - R²、Adjusted R²
    - F 检验结果
2. **检验 1：线性关系（Linearity）**

```r
std_predicted <- as.numeric(scale(fitted(model)))
std_residuals <- rstandard(model)

plot(std_residuals, std_predicted,
     main = "Scatter Plot")
```

+ `fitted(model)`：模型预测值 Y^。
+ `scale(...)`：把预测值标准化（减均值、除标准差），变成 Z 分数。
+ `as.numeric(...)`：把结果转成数值向量。
+ `rstandard(model)`：计算**标准化残差**（residual / 估计标准差），方便比较大小。
+ `plot(std_residuals, std_predicted)`：
    - X 轴：标准化残差
    - Y 轴：标准化预测值 你在看：残差是否随机分布、有没有明显的模式（弯曲、趋势）。 如果是随机云状 → 线性关系假设基本 OK。
3. **检验 2：残差正态性（Normality of Residuals**）

```r
shapiro_test <- shapiro.test(residuals(model))
print(shapiro_test)
```

+ `residuals(model)`：取出模型残差 ei=yi−y^i。
+ `shapiro.test(...)`：做 Shapiro-Wilk 正态性检验。
+ 原假设 H₀：残差服从正态分布。
+ 如果 p > 0.05 → 没有证据拒绝正态性； p 很小 → 残差可能偏离正态。
4. **检验 3：方差齐性（Homoscedasticity）**

```r
bp_test <- bptest(model)
print(bp_test)
```

+ `bptest(model)`：Breusch-Pagan 检验。
+ 检查残差方差是否恒定（不随 X 改变）。
+ H₀：方差齐性（homoscedastic）。
+ p > 0.05 → 没证据说明异方差； p 很小 → 可能存在异方差（heteroscedasticity）。
5. **检验 4：异常值（Outliers）**

```r
std_residuals <- rstandard(model)
threshold_resid <- 1.5 
Outlier_Resid <- abs(std_residuals) > threshold_resid
print(Outlier_Resid)
summary(Outlier_Resid)
```

+ 再次计算标准化残差。
+ `threshold_resid <- 1.5`：你这里设了一个比较宽松的阈值（通常 2 或 3 更常见）。
+ `abs(std_residuals) > threshold_resid`：
    - 返回一个 TRUE/FALSE 向量，TRUE 表示该点是“可疑异常值”。
+ `summary(Outlier_Resid)`：
    - 统计有多少 TRUE / FALSE，即有多少点被标记为异常。
6. **检验 5：残差独立性（Independence）**

```r
install.packages("car")
library(car)
dw_test <- durbinWatsonTest(model)
print(dw_test)
```

+ `durbinWatsonTest(model)`：Durbin-Watson 检验。
+ 检查残差是否存在自相关（尤其是时间序列数据）。
+ DW 值：
    - 接近 2：残差独立性良好。
    - 远小于 2：正自相关。
    - 远大于 2：负自相关。
+ 一般还看 p 值：p > 0.05 → 没证据说明自相关严重。
7. **多重线性回归模型**

```r
#model <- lm(FINAL ~ EXAM1+EXAM2+EXAM3, data = assignmentnew)
model <- lm(height ~ age+wt, data = clinical_trial)
summary(model)
```

+ 这里换成了**多元线性回归**：
    - 因变量：`height`
    - 自变量：`age` 和 `wt`
+ `summary(model)`：同样输出系数、p 值、R²、F 检验等，只是现在有多个自变量。
8. **多元异常值：马氏距离（Mahalanobis Distance）**

```r
X_data <- model.matrix(model)[, -1] 

mah_dist <- mahalanobis(x = X_data, 
                        center = colMeans(X_data),
                        cov = cov(X_data))

print(mah_dist)

max_md <- max(mah_dist)
print(max_md)
```

+ `model.matrix(model)`：提取模型中的设计矩阵（X），包括截距列。
+ `[, -1]`：去掉第一列截距（Intercept），只保留自变量。
+ `mahalanobis(...)`：
    - 计算每个观测点在多维 X 空间中的“距离”，考虑协方差结构。
    - 距离越大 → 该点越“异常”（多元离群点）。
+ `max(mah_dist)`：找出最极端的那个点。
+ 通常会把马氏距离和对应自由度的卡方分布临界值比较，判断是否为异常点。
9. **多重共线性诊断：VIF & Tolerance**

```r
install.packages("car")
library(car)

vif_values <- vif(model)
print(vif_values)

tolerance_values <- 1 / vif_values
print(tolerance_values)
```

+ `vif(model)`：计算每个自变量的 VIF（方差膨胀因子）。
    - 常用经验：
        * VIF > 5：有点严重
        * VIF > 10：严重共线性
+ `1 / vif_values`：手动算 Tolerance（容忍度）。
    - Tolerance = 1 / VIF
    - 越小越糟，一般 < 0.2 认为有共线性问题
