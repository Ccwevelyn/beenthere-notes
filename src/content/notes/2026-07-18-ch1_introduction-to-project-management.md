**PM** - “项目的指挥官”，负责<u>整体战略</u>和<u>资源协</u>调。

 项目管理关注的是**整个项目的生命周期**，不仅仅是软件开发本身：  

+ **<font style="background-color:#FBDE28;">满足所有相关方</font>**<font style="background-color:#FBDE28;">：不仅是用户，还包括客户、管理层、投资人等</font>。  
+ **协调目标**：与不同利益相关者沟通，确保项目目标明确并达成一致。  
+ **范围更广**：软件开发只是整个项目的一部分，**PM还要处理市场、法务、运营等其他方面**。

**SE** -“技术执行者”，把项目目标转化为可运行的软件产品  

+ **满足用户需求，与用户沟通**：聚焦最终**<font style="background-color:#FBDE28;">用户</font>**的功能、体验和性能要求。  
+ **受限资源下工作**：<u><font style="background-color:#FBDE28;">在PM分配的时间</font></u>、人力和预算内完成任务  
+ **向PM提供技术细节**：如**开发**所需时间、技术难点、风险评估等。  

---

| **维度** | **需求分析（Requirement Analysis）** | **整体规划（Project Planning）** |
| --- | --- | --- |
| **目的** | 明确用户或业务的功能需求 | 制定实现这些需求的执行方案 |
| **核心问题** | 我们要开发什么？**用户需要什么**？ | 我们怎么开发？资源怎么分配？ |
| **输出成果** | 需求规格说明书（SRS）、用例图、用户故事等 | 项目计划书、时间表、资源分配表、风险清单等 |
| **参与角色** | 产品经理、业务分析师、软件工程师、用户 | 项目经理、技术负责人、财务人员等 |
| **关注点** | 功能、性能、约束、用户体验 | 时间、成本、人力、风险、沟通机制 |
| **所属阶段** | 软件工程中的“分析阶段” | 项目管理中的“规划阶段” |


---

1. **Initiating（启动阶段）**：
+ 目标：确定项目是否值得做，并获取批准 - 立项
2. **Planning（规划阶段）：**
+ 详细的执行方案，确保资源、时间、预算等安排合理。
+ 主要活动：
    - 项目计划书 Project Plan
    - 明确范围、时间表、预算、质量标准
    - 风险评估与应对策略设计
    - 规划沟通机制和团队分工
3. **Executing（执行阶段）**
+ 按照计划推荐项目
4. **Monitoring & Controlling（监控与控制）**
+ 及时发现错误并纠正偏差
5. **Closing（收尾）**
+ 正式结束项目，总结经验，交付成果。

---

| **PM（项目经理）** | 时间、成本、资源、风险、交付节点 | 通常先制定项目框架 |
| --- | --- | --- |
| **SE（系统工程师）** | 技术架构、系统性能、实现路径 | 在PM框架下进行技术规划 |

##### Why start a project ?
1. **Market** **Demand**
+ 由**大量用户或行业趋势**驱动
    - Fuel shortage -> save energy -> efficient car
    - Demand for online food delivery -> Food delivery platform
2. **Strategic opportunity / Business Need**
    - New APP -> increase revenue
3. **Customer** **Request**
+  由**具体客户或用户群体**提出的明确需求，通常是定制化或针对性强的项目启动动因。 
4. **Technological Advance**
    - Implement Generative AI
5. **Legal** requirement 
    - Government regulation requires ...
        * A new security software program
6. **Social needs**
+ 源于社会整体的福祉、健康、环境或公共安全等方面的需求，通常具有**公益性或政策导向**。  
    - Disease -> Vaccine
7. **Environmental Considerations**
    - Old air conditioners -> energy-efficient one
    - 排放污水前 remove waste substance

1. Underestimated

    2. Cost
    3. Complexity
    4. Schedule ( Timeliness to market)
2. Lack
    1. Planning
    2. Central architecture -  **no unified structural framework** guiding how projects are planned, executed, monitored, and delivered across an organization or team .

    4. Risk management
    5. Stakeholder engagement
    6. Progressive elaboration
    7. Staff training

    3. Leadership
    4. Planned and managed
    5. Staff morale 士气

 项目启动时需要考虑的关键要素：

+ **项目工作说明书（SoW）**：描述将要交付的产品或服务，通常是<font style="color:#2F4BDA;">概括性</font>的，不是详细设计。
+ **商业论证（Business Case）**：说明项目的动机，包括：
    - 约束条件：范围、时间、成本
    - 所需资源：人力、资金、物理资源
    - 预期收益：财务、战略等
+ **协议（Agreements）**：适用于外部项目，明确合作条款。
+ **干系人（Stakeholders）**：包括项目发起人、项目经理及所有对项目有利益关系的人。

| 阶段 | 名称 | 作用 |
| --- | --- | --- |
| P1 | 启动（Initiating） | 明确项目**目标**、授权项目 |
| P2 | 规划（Planning） | 制定**详细**计划，包括范围、时间、成本等 |
| P3 | 执行（Executing） | 实际开展项目工作，交付成果 |
| P4 | 监控与控制（Monitoring & Controlling） | 跟踪进度、调整偏差、确保质量 |
| P5 | 收尾（Closing） | 完成项目、总结经验、交付成果 |


项目的最终输出可以是：

+ **产品/服务**
+ **过程改进**
+ **结果（如报告、数据）**

此外，**经验教训（Lessons Learned）** 是项目收尾阶段的重要产出，有助于未来项目优化。

| 集成管理 Integration | 项目整体协调 |
| --- | --- |
| 范围管理 Scope | 明确和控制项目内容 |
| 时间管理 Schedule | 制定和控制进度 |
| 成本管理 Cost | 预算与成本控制 |
| 质量管理 Quality | 确保交付成果符合标准 |
| 资源管理 Resources | 人力与物理资源分配 |
| 沟通管理 Communiction | 信息传递与反馈机制 |
| 风险管理 Risk | 识别与应对潜在问题 |
| 采购管理 Procurement | 外部资源获取与合同管理 |
| 干系人管理 Stakeholder | 维护与协调利益相关者关系 |

创建项目组合（project portfolios）-  战略性、系统化的管理方式  

项目组合管理（Project Portfolio Management, PPM）是指：从战略角度出发，对组织内所有项目进行统一规划、筛选、排序、资源分配和监控，以实现整体最大价值。

<details class="lake-collapse"><summary id="u5875762a"><span class="ne-text"> 为什么项目组合能提高成功率？  </span></summary><ol class="ne-ol"><li id="u2bad2e61" data-lake-index-type="0"><span class="ne-text">战略对齐（Strategic Alignment）</span></li></ol><p id="ub53c0629" class="ne-p"><span class="ne-text" style="font-size: 13px">确保每个项目都与组织的长期目标一致，避免资源浪费在“方向错了”的项目上。</span></p><ol start="2" class="ne-ol"><li id="u63421f6e" data-lake-index-type="0"><span class="ne-text">优先级排序（Prioritization）</span></li></ol><p id="u86a1cc9c" class="ne-p"><span class="ne-text" style="font-size: 13px">通过评估项目的价值、风险、资源需求，合理安排先做什么、后做什么。</span></p><ol start="3" class="ne-ol"><li id="uba294687" data-lake-index-type="0"><span class="ne-text">资源优化（Resource Optimization）</span></li></ol><p id="u26177964" class="ne-p"><span class="ne-text" style="font-size: 13px">统一管理人力、财力、技术资源，避免重复投入或资源冲突。</span></p><ol start="4" class="ne-ol"><li id="ufd5404cf" data-lake-index-type="0"><span class="ne-text">风险平衡（Risk Balancing）</span></li></ol><p id="uc0be75e5" class="ne-p"><span class="ne-text" style="font-size: 13px">组合中可以混合高风险创新项目与低风险稳定项目，整体风险更可控。</span></p><ol start="5" class="ne-ol"><li id="ub6f82e08" data-lake-index-type="0"><span class="ne-text">持续监控与调整（Continuous Monitoring &amp; Adjustment）</span></li></ol><p id="u8f04d1aa" class="ne-p"><span class="ne-text" style="font-size: 13px">项目组合不是一成不变的，而是根据市场、反馈、资源变化不断优化。</span></p><ol start="6" class="ne-ol"><li id="u9eb3400b" data-lake-index-type="0"><span class="ne-text">投资回报最大化（Value Maximization）</span></li></ol><p id="u602802af" class="ne-p"><span class="ne-text" style="font-size: 13px">通过科学筛选和资源配置，组织能把钱和时间花在最值得的项目上。</span></p></details>
