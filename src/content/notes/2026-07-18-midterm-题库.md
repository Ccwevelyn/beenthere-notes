1. <font style="color:rgb(0,0,0);">Describe</font><font style="color:rgb(0,0,0);background-color:#FBDE28;"> </font>**<font style="color:rgb(0,0,0);background-color:#FBDE28;">three</font>**<font style="color:rgb(0,0,0);background-color:#FBDE28;"> populations</font><font style="color:rgb(0,0,0);"> of users with </font>**<font style="color:rgb(0,0,0);">special needs</font>**<font style="color:rgb(0,0,0);">. For each of these populations, suggest </font>**<font style="color:rgb(0,0,0);">three</font>**<font style="color:rgb(0,0,0);"> ways current interfaces </font>**<font style="color:rgb(0,0,0);">could be improved to</font>**<font style="color:rgb(0,0,0);"> better serve them(O). </font>

**<u>Users with Hearing Impairments</u>**** 损伤**

_Challenges：_

+ May struggle with video content **lacking captions.**  
+ Cannot access audio cues, spoken instructions, or alerts.  

_Improvement：_

+ **Provide accurate captions and transcripts** for all audio and video content.
+ **Use visual alerts** (e.g., flashing icons, banners) instead of sound notifications.
+ **Enable sign language avatars or overlays** 



**<u>Users with Visual Impairments（e.g. Color-blind User）</u>**

_Challenges:_

+  Difficulty distinguishing certain color combinations (e.g., red-green, blue-yellow).  
+   Low contrast between elements can hinder readability and interaction.   

_Interface Improvements_:

+ **Avoid using color as the only means of communication:  **Always pair color with text labels, icons, or shapes.  
    - Example: Instead of showing a red error message alone, include a warning icon and the word “Error.”  
+ **Use color-blind safe palettes  ** Choose combinations that are distinguishable for most types of color blindness (e.g., blue-orange, purple-yellow).  
    -  Avoid problematic pairs like red-green or green-brown.  
+ **Offer color mode toggles or simulation tools**  Provide a "color-blind friendly mode" in settings.  



**<u>Users with Cognitive Disabilities</u>**** **(e.g., dyslexia 阅读障碍, memory challenges)

_Challenges:_

+ May struggle with complex layouts, jargon, or multi-step processes.
+ Need more time and clarity to process information.

_Interface Improvements:_

+ **Use plain language and consistent terminology** to reduce cognitive load.
+ **Break tasks into small, manageable steps** with clear progress indicators.
+ **Offer customizable interfaces** (e.g., simplified mode, reading assistance, icon-based navigation).



 **<u>Older Adults</u>**  

_Challenges:_

+ **Declining vision**: Difficulty reading small text or low-contrast elements.
+ **Cognitive load**: Slower information processing, memory limitations, and unfamiliarity with digital conventions.

_Improvements:_

+ **Use large, readable text and high contrast**
+ **Simplify task flows and reduce steps：**Break complex processes into clear, linear steps.
+ **Design large, clearly labeled buttons：**Avoid relying on icons alone — pair with text.



 **<u>Users with Motor Disabilities</u>**  (运动障碍用户）

_Challenge: _

+ Inability to **precisely operate **a mouse, touchscreen, or keyboard  

_Improvements:_

+ Support **voice control**,or **switch-based input**.开关控制
+ Provide **large buttons** and **sufficiently sized touch targets**.
+ Avoid **time-limited interactions** (e.g., countdown-based verification codes).

3. <font style="color:rgb(0,0,0);">Why is </font>**<font style="color:rgb(0,0,0);">recognition</font>**<font style="color:rgb(0,0,0);"> better than </font>**<font style="color:rgb(0,0,0);">recall</font>**<font style="color:rgb(0,0,0);"> in </font>**<font style="color:rgb(0,0,0);">minimizing the memory load </font>**<font style="color:rgb(0,0,0);">of the user? How do </font>**<font style="color:rgb(0,0,0);">UX designers</font>**<font style="color:rgb(0,0,0);"> minimize the need for </font>**<font style="color:rgb(0,0,0);">recall</font>**<font style="color:rgb(0,0,0);"> in UX Design? </font>

<font style="color:rgb(0,0,255);">https://www.interaction-design.org/literature/topics/recognition-vs-recall?srsltid=Afm BOooOM7jWOsQt62ix41JqfXtcg2zGPEjd9ZR5aKTo5XUlmSwzl18 </font>

Recognition and recall describe **how users retrieve information**. 

+ **<font style="color:#DF2A3F;">Recognition</font>** is the ability to recognize something you **have seen before.**
    - 看到后认出
+ Recall is the ability to **remember** something **without being prompted**. 
    - 听到描述后努力回忆

| **情境** | **Recognition（识别）** | **Recall（回忆）** |
| --- | --- | --- |
| 登录界面 | 显示用户名列表供用户选择 | 要求用户手动输入用户名 |
| 菜单导航 | 显示图标和标签供用户点击 | 要求用户记住命令并输入（如 CLI） |
| 密码恢复 | 提示安全问题供用户选择答案 | 要求用户记住并输入旧密码 |


_Answer_：

**<u>Why</u>**：

**Recognition requires less cognitive effort**: Users can identify familiar elements when they see them, rather than having to retrieve information from memory.  

**Recall requires more effort and can be more prone to errors**：When users rely on their memory to find what they are looking for, they are more likely to become frustrated and abandon the task altogether.

**<u>How</u>**：

**Use familiar icons and labels:** Use standard icons and labels that users can quickly recognize. 

For example, a magnifying glass [icon](https://www.interaction-design.org/literature/topics/iconography) for search or a home icon for the homepage can help users find what they are looking for more easily.

**Provide **[**visual cues**](https://www.interaction-design.org/literature/topics/visual-cues)**: **Visual cues such as [color](https://www.interaction-design.org/literature/topics/color), contrast, and [typography](https://www.interaction-design.org/literature/topics/typography) can help guide users to important information. Use different colors or font sizes for headings, subheadings, and body text to help users scan content more easily.

**Follow common patterns:** Design interfaces that follow common patterns to reduce the need for recall. 

For example, users expect to find a logo in the top-left corner of a website. 

**Group related information together: **This can help users find what they are looking for without the need to remember where it is located. 

For example, grouping all account settings under one tab or menu item can make [navigation](https://www.interaction-design.org/literature/topics/navigation) easier for users.

4. <font style="color:rgb(0,0,0);">Describe at least </font>**<font style="color:rgb(0,0,0);">three different types</font>**<font style="color:rgb(0,0,0);"> of </font>**<font style="color:rgb(0,0,0);">expert review methods</font>**<font style="color:rgb(0,0,0);">. (Ch5)</font>
+ **<font style="color:rgb(0,0,0);">Heuristic evaluation </font>**
+ **<font style="color:rgb(0,0,0);">Guidelines review </font>**
+ **<font style="color:rgb(0,0,0);">Consistency inspection </font>**
+ **<font style="color:rgb(0,0,0);">Cognitive walkthrough </font>**
+ **<font style="color:rgb(0,0,0);">Formal usability inspection</font>**

---

**<u>Heuristic Evaluation</u>****（启发式评估）**

专家根据一组通用的**可用性原则**检查界面是否存在问题。

+ 快速、低成本
+ 适合早期设计阶段

优点：覆盖面广，适合发现常见问题 

缺点：**依赖专家经验**，可能忽略特定用户群体问题

**<u>Guidelines Review</u>****（设计指南审查）**

专家将界面与**官方或行业标准**进行对照检查。【很多设计标准（如 WCAG、Apple HIG、Material Design）并不是“非黑即白”的规则，而是**带有模糊性和上下文依赖**。】

优点：确保界面符合技术标准和法规  

缺点：可能忽略用户实际行为或情境差异

**<u>Consistency Inspection</u>****（一致性检查）**

检查界面内部是否在术语、布局、交互方式上保持一致。

**<u>Cognitive Walkthrough</u>****（认知走查）**

专家模拟新用户完成具体任务的过程，分析每一步是否容易理解和执行。

+ 关注“第一次使用者”的学习路径

**<u>Formal Usability Inspection</u>****（正式可用性审查）**

由多个专家组成评审小组，按照结构化流程评估界面可用性。

优点：系统性强，适合正式报告 

缺点：成本高、流程复杂

6. <font style="color:rgb(0,0,0);background-color:#FBDE28;">What is usability testing? </font><font style="color:rgb(0,0,0);">（O)</font>

Usability testing is a method of **evaluating a product** like a website or app **by observing ****<u>real users</u>** as they **complete** **tasks** to identify problems and opportunities for improvement. 

The goal is to make the product more user-friendly and to ensure it meets user needs by **gathering direct feedback** on the user experience. 

This is often done through a combination of **observing** participants, **collecting** both qualitative (user comments) and quantitative (task success rates) data, and then analyzing the results to inform design decisions.

7. <font style="color:rgb(0,0,0);">“</font><font style="color:rgb(0,0,0);background-color:#FBDE28;">Usability testing is better than expert reviews.” Do you agree?</font><font style="color:rgb(0,0,0);">（O)</font>

<font style="color:rgb(0,0,0);">落脚点在</font>**<font style="color:rgb(0,0,0);">阶段+需求+钱</font>**

| <font style="color:rgb(0,0,0);"></font> | **<font style="color:rgb(0,0,0);">Usability Testing</font>** | **<font style="color:rgb(0,0,0);">Expert Reviews</font>** |
| --- | --- | --- |
| **Perspective** | <font style="color:rgb(0,0,0);">Real users performing real tasks</font> | <font style="color:rgb(0,0,0);">Experts applying heuristics and experience</font> |
| **Strengths** | <font style="color:rgb(0,0,0);">Reveals actual user behavior and pain points</font> | <font style="color:rgb(0,0,0);">Identifies design rule violations and consistency issues</font> |
| **Cost & Speed** | <font style="color:rgb(0,0,0);">Slower, more expensive</font> | <font style="color:rgb(0,0,0);">Faster, cheaper</font> |
| **Depth of Insight** | <font style="color:rgb(0,0,0);">Rich behavioral data, emotional reactions</font> | <font style="color:rgb(0,0,0);">Systematic, rule-based diagnosis</font> |
| **Best For** | <font style="color:rgb(0,0,0);">Validating usability with target audience</font> | <font style="color:rgb(0,0,0);">Early-stage design critique or budget-limited projects</font> |


 

<u>When Usability Testing Is Better</u>？

+ **Need real user feedback** on task completion, confusion points, or emotional reactions.
+ **Testing a live or high-fidelity prototype** and need to validate assumptions.
+ **Targeting specific user groups** (e.g., elderly, children, professionals) whose behavior may differ from expert expectations.

<u>When Expert Reviews Are Better</u>？

+ **In early design stages** and want fast, low-cost feedback.
+ **Need to catch heuristic violations** like inconsistent layouts
+ **Under time or budget constraints** and can't recruit users.

 An interface metaphor is a design technique that uses **familiar concepts from the real world** to make computer interfaces intuitive.

For example:

+ The file and folder analogy for the** file system** of an operating system
+ The trash can icon represents "delete"
+ The "shopping cart" for e-commerce
+ **Whiteboard metaphor**: Collaborative drawing apps

13. <font style="color:rgb(0,0,0);">(O) </font><font style="color:rgb(0,0,0);background-color:#FBDE28;">What is the relationship between </font>**<font style="color:rgb(0,0,0);background-color:#FBDE28;">interface metaphor</font>**<font style="color:rgb(0,0,0);background-color:#FBDE28;"> and </font>**<font style="color:rgb(0,0,0);background-color:#FBDE28;">direct manipulation</font>**<font style="color:rgb(0,0,0);">? (Ch7)</font>

**Interface metaphors** provide the conceptual **foundation** for direct manipulation by making virtual actions feel familiar and intuitive, 

While **direct manipulation** **operationalizes** those metaphors through visible, reversible, and physical interactions.

**<u>Direct Manipulation: Interaction Style</u>**

+ Direct manipulation refers to interfaces where u<u>sers interact with visible objects using </u>**<u>physical actions</u>** (e.g., dragging, clicking, resizing).
+ Key attributes:
    - **Continuous visual representation** of objects
    - **Physical actions** replace complex commands
    - **Immediate feedback** and **reversible operations**

This style **relies on metaphors** to make interactions **feel natural.** For example, dragging a file to the trash bin is meaningful only because the metaphor makes 'delete' feel like 'throwing away.'
